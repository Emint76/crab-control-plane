# SYSTEM_MAP

## Main flow

`intake -> delegation -> validation -> placement`

## Placement targets

- Notion for workflow state
- Obsidian for semantic notes
- KB for sanctioned assets
- Observability for runs, logs, evals

## Runtime relationship

The runtime orchestrator uses:
- policy docs
- packet contracts
- runtime templates
- retrieval discipline

It does not treat Notion or Obsidian as interchangeable storage.
