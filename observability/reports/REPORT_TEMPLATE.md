# REPORT_TEMPLATE

## Scope
State which runs, time window, artifacts, or review decisions the report covers. Include the identifiers used to connect the report back to observability records.

## Findings
Summarize the main observed outcomes, such as repeated routing mistakes, review failures, schema mismatches, or quality improvements.

## Risks
List the operational, provenance, or policy risks exposed by the findings. Distinguish blocking risks from watch items.

## Recommended next steps
List bounded follow-up actions such as prompt revision, schema tightening, additional review, or policy clarification. Do not treat recommendations as self-applying changes.
